Ontology Specification Draft

Abstract

The Football Ontology is used to model the world of football and external factors such as GDP and World Health Organization health statistics for each country in the world. This ontology was created as a part of a group project for the module CS7IS1: Knowledge and Data Engineering, taught in Trinity College Dublin by Professor Declan O’Sullivan. The football ontology models various aspects of football, including but not limited to players, clubs, competitions, player appearances in football games, competitions, football leagues and football transfer mechanics. The football ontology also models various factors regarding a country, such as its GDP and Health Statistics. The Health Statistics for each country modelled by this ontology include obesity rate among children, life expectancy, assistance to medical research, total alcohol consumption, total population and more.

Introduction back to ToC

The football ontology models a world of football while considering external factors of a country such as its GDP and health statistics. The football ontology was created so that we could relate the football world with external factor affecting this sport. By doing so this ontology allows a user to take a deep dive into the mechanics of football and see how external factors such as GDP may affect the decisions of player, clubs and competitions. The goal for this ontology is to model in more areas of football and external factors that affect the world of football. Another goal is to make this ontology more robust.

The Football Ontology: Description back to ToC

This diagram of the football ontology is as follows:

The football ontology has one super class known as Countries. Countries has 3 subclasses known as GDP, WHO and Football. The GDP class is a class for GDP of each country. The WHO class is a class for the health statistics of each Country and the Football class is a class for the football structure in a given country.
The football class contains eight subclasses which are Football Transfers, Games, Appearances, Clubs, Competitions and Players. Each of these eight subclasses are interrelated to each other and have object relationships. For example the relation hasAppeared is between players and appearances, a player hasAppeared in appearances. Each class and subclass also has several data properties attached with them. From the diagram we can see above that the class GDP has an object property known as gdpValue which is the indicator of the GDP value of each country. Each object property and data property has a range and domain attached as well which can be observed from the figure above.
Please reach out to the authors of this ontology in case you may have any questions.

SPARQL Queries back to ToC

This section contains details and explanations about the SPARQL Queries used to query the uplifted datasets that use the football ontology as a prefix in them.

SPARQL Query 1

The following query helps retrieve the list of professional footballers from first class leagues that come from coutries with a GDP greater than the median GDP of all countries

The SPARQL Query:

prefix football: <http://www.cs7is1.org/ontologies/football-ontology-3#>

      select ?players_entity ?country ?country_entity ?gdpValue
      where {
          ?competiton_entity football:type ?tier;
                             football:competition_id ?competition_id .
          ?club_entity football:domestic_competition_id ?competition_id;
                       football:club_id ?club_id .
          ?players_entity football:current_club_id ?club_id;
                          a football:Players;
                          football:country_of_birth ?country.
          ?country_entity football:country ?country;
                          football:gdpValue ?gdpValue
          FILTER(?tier = "first_tier" && ?gdpValue >7.33E+10) .
      }.
